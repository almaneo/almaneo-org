## 9. 治理与基础

### 权力必须去中心化

AlmaNEO 不是一个由任何个人或组织主导的项目。我们从一开始就设计了去中心化的治理模式，赋予社区决定自身未来的权力。

---

### 9.1 治理理念

#### 三大原则

| 原则 | 说明 |

|------|------|

| **透明度** | 所有决策流程和财务使用详情均在链上公开 |

| **参与性** | 任何代币持有者都可以提议、投票并参与决策 |

| **贡献权重** | 贡献越多的人的声音越响亮，而拥有更多代币的人的声音则越响亮 |

---

### 9.2 DAO 架构

AlmaNEO DAO 汇集社区的集体智慧，共同决定项目的发展方向。

### AlmaNEO DAO 架构

**AlmaNEO DAO 架构：**

| 组成 | 角色 | 描述 |

|:---|:---|:---|

| **代币持有者** | 全体社区成员 | 提案投票权、善意加权投票、理事会选举 |

| **理事会** | 理事会（9 名成员） | 紧急议程处理、提案预审 |

| **基金会** | 基金会 | 日常运营、法律代表、DAO 决策执行 |

**关系：** 代币持有者选举理事会 → 理事会与基金会合作 → 基金会支持/执行持有者

---

### 9.3 投票机制

#### 善意加权投票

AlmaNEO 的投票系统并非简单的“1 个代币 = 1 票”。

```

投票权 = √(持有代币数量) × 友善度倍率

```

| 友善度 | 倍率 |

|----------------|------|

| 0 - 100 | 1.0倍 |

| 101 - 500 | 1.5倍 |

| 501 - 1000 | 2.0倍 |

| 1001+ | 2.5倍 |

**此系统的意义：**

- 限制鲸鱼（持有大量代币者）的垄断权力

- 增强活跃社区贡献者的话语权

- 建立友善度等于影响力的机制

#### 提案类型

| 类型 | 法定人数 | 通过要求 | 示例 |

|------|--------|----------|------|

| **通用提案** | 5% | 多数 | 小预算使用，合作 |

| **重要提案** | 15% | 三分之二 | 博览会主办、大型资助 |

| **章程修正案** | 30% | 四分之三 | 代币经济变革、治理结构变革 |

---

### 9.4 AlmaNEO 基金会

基金会是负责在现实世界中执行 DAO 决策的法律实体。

#### 角色

| 角色 | 描述 |

|------|------|

| **法定代表人** | 合同执行、合作关系、监管响应 |

| **日常运营** | 开发团队管理、基础设施运营 |

| **DAO 执行** | 执行社区决策 |

| **资产管理** | 基金会储备金管理（多重签名） |

#### 透明度承诺

- **季度财务报告** 披露

- **所有支出** 均在链上记录

- **年度外部审计** 已完成

- **理事会监督** 已接受

#### 基金会设立计划

基金会的选址将根据以下因素确定：

| 考虑因素 | 说明 |

|----------|------|

| **DAO友好型法律** | 法律认可DAO活动的司法管辖区 |

| **税务效率** | 适用于非营利活动的税收政策 |

| **运营便捷性** | 可远程设立/运营 |

| **声誉** | 合法的创新区，而非规避监管区 |

> *具体选址将在法律审核后最终确定并公布。*

---

### 9.5 逐步去中心化

AlmaNEO 并非从一开始就追求完全去中心化。快速去中心化实际上可能会给项目带来风险。

### 逐步去中心化路线图

| 阶段 | 周期 | 特点 | 主要内容 |

| :--- | :--- | :--- | :--- |

| **第一阶段** | 2025-2026 | 基金会主导 | 建立核心基础设施，组建初始社区，基金会做出重大决策 |

| **第二阶段** | 2026-2027 | 联合治理 | 开始理事会选举，DAO 对重大议程进行投票，并逐步将权力移交给基金会 |

| **第三阶段** | 2027-2028 | DAO 主导 |大多数决策由去中心化自治组织 (DAO) 做出，基金会将过渡到执行机构 |

| **第四阶段** | 2028 年及以后 | 完全去中心化 | 基金会规模最小化或解散，基于智能合约的自动化运营，完全社区自治 |

---

### 9.6 安全保障

#### 紧急暂停

如遇严重安全威胁或漏洞，经理事会多数投票（5/9）可暂时中止协议运行。

协议将在中止后 48 小时内通过 DAO 投票恢复或解决。

#### 多重签名

基金会资产通过多重签名钱包进行管理。

| 用途 | 签名要求 |

|------|----------|

| 日常运营支出 | 3/5 签名 |

| 大额支出 | 5/7 签名 |

| 应急基金 | 7/9 签名 + DAO 投票 |

#### 时效锁定

智能合约的重大变更将受到 48 小时时效锁定，以确保社区有时间进行审查并提出异议。
